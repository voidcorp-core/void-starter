# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in void-starter, please report it privately:

1. Email **security@volpio.com** with the subject line `void-starter security: <short description>`.
2. Include a clear description, reproduction steps, and impact assessment.
3. Do not file a public GitHub issue for security-sensitive reports.

We aim to acknowledge receipt within 48 hours and to ship a fix within 14 days for critical issues.

## Scope

This policy covers the void-starter repository (the template foundation). For MVPs derived from this starter, contact the project's own security policy or the Volpio team.

## Technical primitives

For the technical security mapping (OWASP Top 10, RGPD checklist, primitives in this starter), see `docs/SECURITY.md`.

## Supported versions

The starter is single-track: only the latest tag is supported. There is no LTS branch. If you need a fix backported, open an issue.
